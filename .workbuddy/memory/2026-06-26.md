- **背景**：确认 agent-conch 需支持 skill 加载（SKILL.md 规范的按需技能包）
- **决策**：采用方案 A——Skill 系统作为域1（信息边界与指令）的插件实现，而非独立成第10域

- **改动**：`docs/technical-design.md`

## 技术方案 v0.3：v0.2 架构改进建议落地

- **背景**：用户提供 v0.2 架构改进建议（分层倒置/职责重叠、Registry/Profile/Hook 增强、可观测/安全/成本补齐、生态协同），要求复杂度与完备性取舍
- **取舍原则**：约束/边界/基线类立即做（越早定越省事）；企业级功能延后到 L4（越早做越负债）
- **落地改动**（`docs/technical-design.md` 升 v0.3）：
  - 依赖倒置澄清 + 合并独立编排层到域5（消除分层倒置）
  - Hook 三大约束：职责隔离 + priority 优先级 + 中断节点白名单
  - Profile `extends` 继承 + Pydantic 全量校验 + 环境变量覆盖
  - 沙箱加固基线（CPU/内存/网络限制/禁特权/只读挂载）进入 MVP
  - Agent 自观测（核心指标封装只读工具，轻量版）
  - 四级指标集（执行/成本/效果/健康），MVP 先做执行+成本
  - 标准基准对接（SWE-bench/MT-Bench 原生集成）
  - MCP 对齐强化 + 成本分级降级（L1压缩→L2切模型→L4终止，L3延后）
  - OrchestrationMode 三方法接口预留（task_split/state_sync/conflict_resolve，L3实现）
- **明确延后到 L4**：资源池调度、语义化版本约束、签名校验、RBAC、审计哈希防篡改、预算分组、配置中心
- **关键决策**：Manus 五次重写越做越简单是反例警示——一开始做厚后来砍，我们不该重蹈覆辙
